# 1Panel 部署 BEpusdt 完整指南

> 📋 **前置条件**：本教程假定您已成功安装并配置好 1Panel 面板。

## 快速部署指南

### 第一步：创建容器

在 1Panel 控制台中，找到并点击【**创建容器**】按钮。

![创建容器入口](./img/1.png)

---

### 第二步：配置容器参数

![容器配置界面](./img/2.png)

#### 关键配置说明

| 配置项      | 说明                                                                                      |
|----------|-----------------------------------------------------------------------------------------|
| **镜像选择** | • **正式版**：`v03413/bepusdt:latest`（推荐生产环境）<br>• **开发版**：`v03413/bepusdt:nightly`（体验最新特性） |
| **端口映射** | • **本机访问**：`127.0.0.1:8080:8080`（推荐配合反向代理）<br>• **外网暴露**：`8080:8080`（不推荐，存在安全隐患）        |

> ⚠️ **安全建议**：为确保服务安全，建议使用 `127.0.0.1` 绑定本地端口，再通过反向代理对外提供服务。

确认配置无误后，点击【**确认**】按钮启动容器。

---

### 第三步：验证启动状态

容器启动后，点击【**日志**】查看启动日志。

当日志中出现服务监听地址和端口信息时，表示服务已成功启动：

![日志输出 - 启动成功](./img/3.png)

✅ **启动成功标志**：日志显示服务正在监听指定地址和端口

---

## 配置反向代理（推荐）

### 为什么需要反向代理？

- ✅ 启用 HTTPS 安全访问
- ✅ 隐藏真实端口，提升安全性
- ✅ 支持自定义域名，展现专业形象

### 配置步骤

#### 1. 进入网站管理

返回 1Panel 首页 → 左侧菜单【**网站**】→ 点击【**创建网站**】→ 选择【**反向代理**】

> 📌 **前提条件**：确保已安装 OpenResty

#### 2. 填写代理配置

![反向代理配置](./img/5.png)

| 配置项      | 填写示例                    | 说明            |
|----------|-------------------------|---------------|
| **主域名**  | `coin.example.com`      | 填写您的域名        |
| **代理地址** | `http://127.0.0.1:8080` | 指向容器运行端口      |
| **其他设置** | 按需配置                    | 如启用缓存、自定义请求头等 |

#### 3. 完成域名解析

在域名服务商处添加 A 记录，将域名指向服务器 IP。

#### 4. 配置 HTTPS（强烈推荐）

- 在网站管理中申请或上传 SSL 证书
- 启用 HTTPS 强制跳转
- 提升服务安全性和用户信任度

---

## 验证部署

配置完成后，通过浏览器访问您的域名，应能正常访问 BEpusdt 服务。

## 常见问题

**Q：无法访问服务，应如何排查？**

A：请依次检查以下项目：

1. 容器是否正常运行（查看启动日志）
2. 防火墙是否拦截相应端口
3. 域名解析是否生效（执行 `ping your-domain.com` 验证）
4. 反向代理配置是否正确

**Q：是否必须配置反向代理？**

A：不是必须配置，但我们强烈推荐。直接暴露容器端口存在安全风险，且无法使用 HTTPS 加密传输。
